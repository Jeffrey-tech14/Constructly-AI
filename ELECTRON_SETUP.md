# Electron Desktop App Setup

This guide explains how to develop and build the Constructly desktop application using Electron.

## Project Structure

```
electron/
  ├── main.ts          # Electron main process (handles window creation, menus, etc.)
  └── preload.ts       # Preload script (secure IPC bridge between renderer and main)

src/
  ├── electron.d.ts    # TypeScript declarations for Electron API
  ├── App.tsx
  ├── main.tsx         # React app entry point (works for both web and Electron)
  └── ...

dist/                  # Built web app (generated by Vite)
dist_electron/         # Built Electron app (generated by electron-builder)
```

## Development

### Prerequisites

- Node.js 16+ or Bun
- npm or Bun package manager

### Install Dependencies

```bash
npm install
# or
bun install
```

### Run Development Mode

Start both the Vite dev server and Electron in one command:

```bash
npm run dev:electron
```

This will:

1. Start Vite dev server on http://localhost:8080
2. Wait for the server to be ready
3. Launch Electron app pointing to the local dev server
4. DevTools will open automatically for debugging

Your web app will reload automatically when you save files. For Electron main process changes, you'll need to rebuild and restart.

## Building

### Build for Current Platform

**Windows:**

```bash
npm run build:electron:win
```

**macOS:**

```bash
npm run build:electron:mac
```

**All Platforms:**

```bash
npm run build:electron:all
```

**Default (auto-detects platform):**

```bash
npm run build:electron
```

This will:

1. Build the web app with Vite (outputs to `dist/`)
2. Package an Electron app using electron-builder
3. Create installers and portable executables in `dist_electron/`

### Build Output

After building, you'll find:

**Windows:**

- `Constructly-Setup-{version}.exe` - NSIS installer
- `Constructly-{version}.exe` - Portable executable

**macOS:**

- `Constructly-{version}.dmg` - Disk image
- `Constructly-{version}.zip` - ZIP archive

## Publishing Updates

### GitHub Releases (Recommended)

1. Update version in `package.json`
2. Push to GitHub
3. Create a GitHub release with the tag

The app automatically checks for updates on startup and notifies users when a new version is available.

### Manual Distribution

Simply distribute the installer or portable executable to users.

## Automatic Updates

The application comes with built-in automatic update checking powered by `electron-updater`.

### How It Works

1. **On Startup**: The app automatically checks GitHub releases for new versions
2. **User Notification**: If an update is available, a dialog appears asking the user to download
3. **Download & Install**: Users can download the update with progress tracking
4. **Restart to Apply**: After download completes, users are prompted to restart to install

### Update Flow

```
App Startup
    ↓
Check GitHub Releases
    ↓
Update Available? → Show Dialog
    ↓
User clicks "Download & Install"
    ↓
Download with Progress
    ↓
Update Downloaded → Show "Install Now" Dialog
    ↓
User clicks "Install & Restart"
    ↓
App Closes & Installer Runs
```

### Configuration

The update checker is configured in `electron-builder.yml`:

```yaml
publish:
  provider: github
  owner: Jeffrey-tech14
  repo: Constructly-AI # Make sure this matches your repo name
```

### Disabling Updates for Development

Updates are automatically disabled in development mode. To test update functionality:

```bash
npm run build:electron:win
# Then run the executable directly (not through npm dev)
```

### Related Files

- `electron/updater.ts` - Update checker implementation
- `src/components/UpdateNotifier.tsx` - Update UI component
- `src/hooks/useElectronUpdater.ts` - React hook for update state
- `electron-builder.yml` - Update configuration

### Testing Updates Locally

To test updates without publishing to GitHub:

1. Manually create test builds with different version numbers
2. Host them on a local server
3. Modify the `electron-builder.yml` to point to your test server

## Configuration

### electron-builder.yml

Key settings:

- `appId`: Unique application identifier (used for updates and registry)
- `productName`: Display name of the app
- `files`: Files to include in the app bundle
- `win`, `mac`, `linux`: Platform-specific settings
- `publish`: Auto-update configuration (requires GitHub setup)

### Adjusting Window Size

Edit `electron/main.ts`:

```typescript
const createWindow = () => {
  mainWindow = new BrowserWindow({
    width: 1400,      // Default width
    height: 900,      // Default height
    minWidth: 1000,   // Minimum width
    minHeight: 700,   // Minimum height
    ...
  });
};
```

### Application Icon

Replace `public/icon.png` (256×256px minimum) to customize the app icon for Windows and macOS.

## IPC Communication

### Exposing APIs to Renderer

To safely expose Electron APIs to your React app:

1. **Add to preload.ts:**

```typescript
contextBridge.exposeInMainWorld("electronAPI", {
  myFunction: () => ipcRenderer.invoke("my-function"),
});
```

2. **Add handler in main.ts:**

```typescript
ipcMain.handle("my-function", () => {
  return "result from main process";
});
```

3. **Use in React component:**

```typescript
const result = await window.electronAPI.myFunction();
```

4. **Add TypeScript declaration in electron.d.ts:**

```typescript
declare global {
  interface Window {
    electronAPI: {
      myFunction: () => Promise<string>;
    };
  }
}
```

## Troubleshooting

### App won't start with dev server

- Ensure port 8080 is available
- Check the terminal for Vite errors
- Wait for "ready on http://localhost:8080" before Electron launches

### Blank white window

- Check browser DevTools (Ctrl+I on Windows, Cmd+Option+I on macOS)
- Verify `preload.ts` and `main.ts` paths are correct
- Ensure `public/icon.png` exists

### Build fails

- Delete `node_modules` and reinstall: `npm install`
- Clear cache: `rm -rf dist dist_electron`
- Check electron-builder.yml configuration

### Distribution/Code Signing (macOS)

For distributing on macOS, you'll need Apple Developer certificates. Configure in electron-builder.yml:

```yaml
mac:
  signingIdentity: "Developer ID Application: Your Name"
  certificateFile: path/to/certificate.p12
  certificatePassword: password
```

## Performance Tips

1. **Minimize main process work** - Offload heavy computation to renderer
2. **Use lazy loading** - Code-split large sections of your React app
3. **Cache dependencies** - The Electron app packages all dependencies; consider bundle size
4. **Memory usage** - Monitor dev tools to identify memory leaks

## Security

- ✅ Context isolation enabled (`contextIsolation: true`)
- ✅ Node integration disabled (`nodeIntegration: false`)
- ✅ Pre-load script for safe IPC
- ✅ Content Security Policy recommended for production

For production, consider adding CSP headers in your app.
